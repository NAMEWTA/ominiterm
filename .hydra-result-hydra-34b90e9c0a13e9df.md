# Hydra Task Result

## Task
Investigate and fix: terminal status stays 'waiting' instead of 'completed' for Claude Code sub-agents spawned via hydra.

## Root Causes Found

### Bug 1: PID lost after process exit (TerminalTile.tsx)
`pollSessionId` calls `getPid(ptyId)` on each poll attempt. But `pty-manager.ts` deletes the pty instance from its map on process exit (line 63). After that, `getPid()` returns `undefined`, and all subsequent poll attempts silently skip the session file lookup. If the Claude agent completes before `pollSessionId` captures the session ID, the ID is never captured and `session.watch()` is never called.

**Fix**: Capture the PID once upfront before the first poll sleep, so it survives process exit. Fall back to live lookup only if the cached PID is null.

### Bug 2: No initial completion check in SessionWatcher.watch() (session-watcher.ts)
`SessionWatcher.watch()` only reacts to future file changes via `fs.watch()`. If `session.watch()` is called after the agent has already completed its turn (the completion signal is already written to the JSONL file), the watcher never fires because no new writes occur. The terminal stays 'waiting' forever.

**Fix**: After setting up the fs.watch watcher, perform an immediate `checkTurnComplete()` on the existing file contents. If the turn is already complete, fire the callback right away.

### `src/terminal/TerminalTile.tsx`

### `electron/session-watcher.ts`
