# i18n Design: Chinese / English Support

## Overview

Add runtime locale switching (zh/en) to the termcanvas Electron app. Use a custom
zero-dependency i18n hook backed by Zustand. Add a Settings modal in the Toolbar
where the user can switch language and theme.

## Requirements

- Runtime switching without app restart
- Default language auto-detected from `navigator.language`
- Locale persisted to `localStorage`
- New Settings modal (gear icon in Toolbar)
- Theme toggle moved from Toolbar into Settings modal

## Architecture

### New files

| File | Purpose |
|------|---------|
| `src/i18n/en.ts` | English string dictionary (TypeScript const object) |
| `src/i18n/zh.ts` | Chinese string dictionary |
| `src/i18n/useT.ts` | `useT()` hook — subscribes to localeStore, returns `t(key)` |
| `src/stores/localeStore.ts` | Zustand store: `locale: 'en' \| 'zh'`, persisted via localStorage |
| `src/components/SettingsModal.tsx` | Modal with Language + Theme toggles |

### Modified files

`src/toolbar/Toolbar.tsx` — remove theme toggle, add gear icon → opens SettingsModal
`src/App.tsx` — use `t()` in CloseDialog
`src/components/Sidebar.tsx` — use `t()` for all UI strings
`src/containers/WorktreeContainer.tsx` — use `t()` for button labels
`src/containers/ProjectContainer.tsx` — use `t()` for "Project" label
`src/components/DiffCard.tsx` — use `t()` for status/label strings
`src/terminal/TerminalTile.tsx` — use `t()` for error messages
`src/toolbar/DrawingPanel.tsx` — use `t()` for tool labels

## Settings Modal Layout

```
┌────────────────────────────┐
│  Settings             ✕   │
├────────────────────────────┤
│  Language  [中文] [English]│
│  Theme     [Dark] [Light]  │
└────────────────────────────┘
```

Style matches existing CloseDialog: `var(--surface)` background, `var(--border)` border,
13px text, same button radius.

## Data Flow

1. App starts → `localeStore` reads `localStorage` → fallback to `navigator.language`
2. Component calls `const { t } = useT()` → reads current locale → returns string
3. User opens Settings → clicks language button → `localeStore.setLocale()` →
   all components re-render with new strings instantly
